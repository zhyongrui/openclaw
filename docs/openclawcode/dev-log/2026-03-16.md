## Blueprint-First Product Direction

Documented the newly clarified product direction instead of leaving it only in
chat:

- users should be able to discuss a project goal with the system before issue
  creation
- the agreed goal should be written into a fixed blueprint document
- the system should later derive or discover work items from that blueprint
- execution stages should support Codex, Claude Code, or both through a shared
  provider-role abstraction
- humans should be able to intervene at every major stage, while the system
  continues autonomously when they do not intervene

Captured that gap analysis in:

- `docs/openclawcode/blueprint-first-orchestration.md`

The key finding recorded there is:

- the current codebase is already strong in the issue-driven execution half
- the major missing pieces are the upstream control plane:
  - blueprint discussion
  - blueprint storage
  - blueprint-to-work-item decomposition
  - general self-discovery
  - provider-neutral multi-agent routing
  - stage-level human handoff

Linked that document into:

- `README.md`
- `development-plan.md`
- `full-program-roadmap.md`
- `master-delivery-checklist.md`
