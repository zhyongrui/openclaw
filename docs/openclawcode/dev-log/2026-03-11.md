## Temporary Public Ingress Automation

Added a small operator-facing workflow for the current constraint where
`openclawcode` cannot use a truly fixed public domain yet and must rely on
anonymous `trycloudflare` URLs.

Implemented:

- `scripts/openclawcode-webhook-tunnel.sh`
  - `start`
    - starts a `cloudflared` quick tunnel
    - waits for the `https://*.trycloudflare.com` URL
    - updates the GitHub webhook URL automatically
  - `run`
    - runs the tunnel in the foreground for dedicated terminals or `tmux`
  - `sync-hook`
    - rewrites the GitHub webhook URL to the current tunnel host
  - `status`, `stop`, `restart`, `print-url`
- `docs/openclawcode/webhook-operations.md`
  - runbook for temporary public ingress mode
  - startup order and operator commands
- `docs/openclawcode/README.md`
  - linked the new webhook operations runbook from the index

Hardening details:

- the script now auto-sources `~/.openclaw/openclawcode.env` by default
- webhook sync writes both:
  - the current quick-tunnel URL
  - the local GitHub webhook secret, when present
- this avoids a regression where rewriting only the URL caused real GitHub
  deliveries to fail signature verification with `401`

Operational note:

- anonymous `trycloudflare` hostnames are still ephemeral
- the new script removes the manual GitHub webhook editing step, but not the
  temporary-domain limitation itself
