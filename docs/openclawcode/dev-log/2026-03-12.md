- the second run reached `ready-for-human-review` and published `PR #47`, but
  exposed two more real issues:
  - draft PR metadata still hard-coded the base branch to `main`, so the PR
    opened against the wrong base and expanded to a large diff
  - deterministic sandbox edits still failed in the live builder, even though
    the builder could recover with a narrow shell rewrite

Root cause and repair:

- reproduced the edit failure directly against the real live sandbox container
  instead of through the agent loop
- confirmed `SandboxFsBridge.writeFile(...)` itself was writing empty files in
  the live container
- traced the bug to the pinned mutation helper shell plan using
  `python3 - <<'PY' ...`, which consumes stdin for the embedded Python script
  and leaves an empty payload for the actual file write
- changed the helper to materialize the embedded Python into a shell variable
  and execute it via `python3 -c`, preserving stdin for the file payload
- fixed draft PR metadata to inherit the workflow workspace base branch instead
  of hard-coding `main`
